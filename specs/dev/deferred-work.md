## Deferred from: code review of story-3.1-3.2-azdo-create-ticket (2026-04-23)

- **Cross-project link-target validation missing.** Skill pre-validates link targets via `wit_get_work_items_batch_by_ids({ ids, project })` scoped to the create's project. AzDO returns items by ID org-wide, so a target from a different project resolves fine but the subsequent `wit_work_items_link` call may fail or create a semantically wrong link. No `target.System.TeamProject == project` check. Defer reason: uncommon flow; Story 3.2 scope is single-project link batches; add when a real cross-project request surfaces.
- **`AZDO_ORG_URL` trailing-slash not normalized.** `config.orgUrl` is documented as "always non-empty" but no normalization â€” `https://dev.azure.com/foo/` produces malformed constructed URLs `â€¦//foo//_workitems/edit/<id>`. Defer reason: operator-controllable; a `.replace(/\/$/, '')` is a one-line fix when the first malformed link appears in the wild.
- **Priority inference reads quoted incident descriptions as urgent.** `/azdo-create-ticket` infers priority from draft content; if the user pastes an incident report as context, the heuristic reads "production down" and picks Priority 1 for what is actually a follow-up investigation ticket. Defer reason: requires a meta-signal ("is this authored or quoted"); real refinement for later.
- **Edit-loop invalidation after approval.** User says "create" then immediately "wait, change title" before the mutation fires. No rule that new edit requests invalidate the prior approval â€” a naive implementation could skip straight to mutate. Defer reason: mutation-confirmation rule edge case; practical risk low because the tool call is issued synchronously on approval.
- **AC "never ship TBD/n/a" has no escape valve.** If the user insists "AC: n/a" the skill says "surface the policy conflict" but names no concrete exit. Defer reason: minor UX deadlock; real behavior in practice is the user rewrites the AC.
- **`wit_work_items_link` with zero-length `updates[]` after edit-loop drops all links.** Ambiguous whether the skill clears the Links-section flag or sends an empty batch. Defer reason: corner case, unambiguous clarification for later revision.
- **`get_azdo_context` tool description doesn't hint that some skills need `team` non-null.** Description says `team` is null-when-unset; skills that require it enforce at the skill layer. Cosmetic doc drift, not a functional gap. Defer reason: tool description stays generic; per-skill constraints live in SKILLs.

## Deferred from: code review of story-1.2-configuration-loader (2026-04-22)

- **Startup-error stack trace on missing env.** Node's default uncaught-exception handler prints `/Users/<user>/â€¦` and full stack on any throw from `src/config.ts` at module evaluation. Fix would either (a) register `process.on('uncaughtException', â€¦)` via a Node `--import` preload module loaded before `src/index.ts`, or (b) move config validation behind an explicit `loadConfig()` call and wrap in `try/catch`. Story 1.3 explicitly rejected both: (a) adds a preload + one file of infrastructure for an aesthetic dev-time error format, (b) changes Story 1.2's committed module-load-time validation contract and every downstream `config.x` access site. The stack trace's first line is already `Error: <KEY> is required`, which is the actionable signal â€” the rest is developer-only noise. Revisit if user-facing stderr formatting becomes a requirement.
- **Placeholder `.env` boots silently.** Literal `<your-org>` / `<your-personal-access-token>` strings are truthy and pass `requireEnv`. Cloning the repo and running `pnpm start` without editing `.env` produces a successful-looking startup that fails opaquely later. Defer reason: NFR-S1 intentionally commits placeholder `.env` for variable discoverability; placeholder-detection is post-MVP hardening.
- **No PAT redaction on `config` object.** `console.log(config)` or any accidental object serialisation leaks the PAT. Defer reason: personal local tool with no shared logs or CI; redaction (custom `inspect` / `toJSON`) is post-MVP.
- **Whitespace in quoted env values preserved.** `AZDO_PAT="  abc  "` survives validation with padding intact, breaks auth silently. Defer reason: input hardening is post-MVP.
- **No `AZDO_ORG_URL` format validation.** Typos, missing scheme, trailing slash pass startup and break downstream as opaque 404/DNS errors. Defer reason: `new URL(value)` probe is post-MVP hardening.
- **Review gate has no mechanical enforcement.** Commit policy lives in `.claude/rules/commit-policy.md`; no pre-commit hook or CI check blocks a commit that skipped review. Defer reason: author-discipline at MVP per scaffold decision; hook/CI is post-MVP.
- **`requireEnv` does not trim before emptiness check.** Couples with whitespace-preservation above. Defer reason: post-MVP.
- **`process.env`-only-in-`config.ts` boundary is unprotected.** Architecture.md:589 mandates the boundary; no ESLint rule or test enforces it â€” a future story can silently violate. Defer reason: lint/test tooling is post-MVP.

## Deferred from: code review of story-2.3-list-team-iterations (2026-04-22)

- **Invalid-`Date` values from upstream produce non-deterministic sort order.** `src/tools/iterations.ts:27` â€” SDK `formatResponse` converts `startDate` strings to `Date`, but a malformed payload (never observed in practice for AzDO) yields `Invalid Date`, which passes the truthy filter at line 25 and returns `NaN` from `.getTime()`. `NaN` comparisons are non-deterministic in V8's sort. Fix is one line (`&& !isNaN(new Date(it.attributes.startDate).getTime())` in the filter). Defer reason: never observed in practice; sort only degenerates for the one malformed entry's position, not the whole list; noise-to-benefit ratio unfavourable at MVP.
- **No upper bound on `limit` parameter.** `src/tools/iterations.ts:17` â€” `z.number().int().positive().default(2)` accepts `Number.MAX_SAFE_INTEGER`. A caller asking for `limit: 100000` forces the full subscription fetch and a giant JSON payload. Defer reason: AC requested no cap; team subscriptions bounded to dozens in practice; a `.max(N)` cap is trivial to add when a real abuse vector surfaces.
- **No tiebreaker for iterations with identical `startDate`.** `src/tools/iterations.ts:27` â€” two iterations sharing a start date compare to `0`, and the top-N slice ordering is not deterministic across calls. Fix is one line (`|| (a.id ?? '').localeCompare(b.id ?? '')`). Defer reason: real-world teams do not run overlapping sprints from the same start date; a tiebreaker is cosmetic hardening, not a correctness fix.
